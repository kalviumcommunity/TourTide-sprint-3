# Source Code Directory

This directory contains the main source code for the TourTide application.

## Purpose
- Core application logic and modules
- TourTide-specific functionality
- Reusable components and classes

## Module Structure

### Core Modules
- `tourist_forecast.py` - Main forecasting engine
- `data_processor.py` - Data handling and processing
- `api_interface.py` - External API connections
- `database_manager.py` - Database operations

### Models
- `models/forecast_model.py` - Prediction model classes
- `models/data_models.py` - Data structure definitions
- `models/config_models.py` - Configuration models

### Utils
- `utils/helpers.py` - Helper functions
- `utils/validators.py` - Input validation
- `utils/calculators.py` - Mathematical operations

## Development Guidelines
- Follow object-oriented design principles
- Implement proper error handling
- Include comprehensive docstrings
- Write unit tests for all modules

## Integration
- Modules should integrate seamlessly
- Use dependency injection patterns
- Maintain clean interfaces
- Provide clear API documentation
